# Feature Notes

Create one subdirectory per feature/topic.

Example:

- `token-capture-determinism/`
- `resolver-lockfile-v1/`

Inside each directory, create one or more date-stamped markdown notes.

## Note Template

### YYYY-MM-DD - <Short Title>

- Commit: <git hash or NO_COMMIT_YET>
- Change summary:
- Motivation:
- Risks/Tradeoffs:
- Validation/Test follow-up:
- Open questions:
