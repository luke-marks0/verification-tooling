# Issue Notes

Create one subdirectory per issue/investigation (cross-cutting or unresolved work).

Example:

- `multi-rack-dispatch-ordering/`
- `hardware-profile-versioning/`

Inside each directory, add date-stamped markdown notes.

## Note Template

### YYYY-MM-DD - <Short Title>

- Commit: <git hash or NO_COMMIT_YET>
- Issue statement:
- Investigation summary:
- Decision/Direction:
- Risks/Tradeoffs:
- Validation/Test follow-up:
