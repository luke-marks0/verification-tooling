# Experiment log: prover-verifier-demo

Started 2026-05-04. See docs/plans/prover-verifier-demo.md.

- 2026-05-04: Task 0.1 — scaffolded experiment directory.
- 2026-05-04: Task 0.2 — confirmed baseline green (321 unit tests, schema gate, 11 fixtures). pydantic missing from system python; created `.venv` via `uv venv .venv` and installed pydantic+jsonschema there. All canonical schema files are single-line sorted-keys (validated by scripts/ci/check_canonical_json.py).
- 2026-05-04: Task 0.3 — wired ruff + pyright + hypothesis (scoped tooling). `make lint-proverdet`, `make typecheck-proverdet`, `make test-proverdet` short-circuit cleanly when the proverdet code/test paths don't exist yet. Engineer's pre-commit habit: run all three before each commit, alongside `make test-fast` and `make schema`.
- 2026-05-04: Task 1.1 — added prover_graph.v1 schema (placeholder). 10 schema tests cover positive/negative cases including unknown fields, missing run_id, sha256-prefixed commitments, and rejecting string-typed claimed_flops.
- 2026-05-04: Task 1.2 — added replay_request.v1 + replay_evidence.v1 schemas. ReplayRequest uses oneOf for {kind: task | artifact} target; ProofOfWork dtype restricted to bf16/fp16/int8. ReplayEvidence carries pow_stream entries with matmul_dim+rounds+dtype duplicated from the request so the verdict engine can compute observed FLOPs from the transcript alone.
- 2026-05-04: Task 1.3 — added verifier_transcript_entry.v1 schema. Mirrors the CaptureLog shape (seq, direction, endpoint, timestamp, payload_digest sha256-prefixed) with optional status_code and payload_path. The verdict engine reads this transcript shape; locking it now keeps later tasks from drifting.
- 2026-05-04: Task 1.4 — added pkg/proverdet/wire.py Pydantic models for Graph, Task, Artifact, Transmission, ReplayRequest (+ TaskTarget/ArtifactTarget discriminated union), ErasureSpec, ProofOfWorkSpec, ReplayEvidence, TranscriptEntry. Top-level wire models expose to_canonical(). Property tests using hypothesis exercise canonical-bytes round-trip is a fixed point for each. Pyright config needed extraPaths=../.. so it could resolve pkg.common imports.
- 2026-05-04: Task 2.1 — added prover server skeleton with /health. Stdlib BaseHTTPRequestHandler + ThreadingMixIn pattern. Lifecycle test spawns subprocess on port 0, reads bound port via fsync'd port file, hits /health, terminates on SIGTERM. test/proverdet/_helpers.py has http_get_json/http_post_json/http_post_bytes used by every later integration test.
- 2026-05-04: Task 2.2 — added GET /graph returning empty placeholder. pkg/proverdet/graph_builder.py.build_empty_graph(run_id) is the pure constructor; the handler validates the body against schema before serialization (500 if mismatch — programmer error).
- 2026-05-04: Task 2.3 — wired POST /replay returning stub evidence. pkg/proverdet/replay.py.stub_evidence builds a schema-valid ReplayEvidence with empty pow_stream and a deterministic sha256 commitment over `b"stub:" + replay_id`. Handler schema-validates the request, then Pydantic-validates, then schema-validates the evidence (defensive: catches programmer drift between Pydantic dump and schema).
- 2026-05-04: Task 2.4 — added prover-side capture log. pkg/proverdet/capture.py.ProverCaptureLog writes monotonic-seq canonical-JSON entries with sha256-prefixed payload digests; threadsafe under 4-thread × 50-entry stress. Wired into ProverState; the handler records both received requests and sent responses.
- 2026-05-04: Task 3.1 — added verifier server with /traffic ingest. Each POST persists bytes verbatim to traffic-<seq>.bin and records a transcript entry. pkg/proverdet/transcript.py.TranscriptLog validates each entry against verifier_transcript_entry.v1 schema before write.
- 2026-05-04: Task 3.2 — factored pkg/proverdet/_jsonl_log.py.JsonlLog. ProverCaptureLog and TranscriptLog now both inherit; subclasses just set SCHEMA_NAME (TranscriptLog validates per append; CaptureLog opts out). All 82 proverdet tests still green. The factor was cheap because both concrete shapes were already in place — design after seeing two, not before.
- 2026-05-04: Task 3.3 — verifier scheduler. pkg/proverdet/scheduler.py.VerifierScheduler is tick-based and seeded. Tests use a fake clock + fake client for reproducibility; the integration test spawns a real prover and waits for both /graph and /replay to land in the verifier transcript with status 200. HttpProverClient is the production wrapper around stdlib urllib. The verifier server starts the scheduler in a daemon thread by default (--no-scheduler for traffic-only tests).
- 2026-05-04: Task 3.4 — verdict CLI stub. cmd/verifier_cli/main.py reads --transcript, calls pkg/proverdet/verdict.emit_verdict (stub returns {"verdict":"unknown","reasons":[]}), writes canonical-JSON --out. Phase 8 will replace the stub with real signals; the CLI signature stays compatible.
