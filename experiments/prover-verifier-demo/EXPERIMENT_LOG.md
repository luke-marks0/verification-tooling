- 2026-05-04: Task 6.1 — replaced stub /replay with Freivalds-backed evidence. pkg/proverdet/attestation_store.py is a threadsafe in-memory dict (lifetimes match the prover process). pkg/proverdet/replay.py.produce_evidence builds a Challenge of `pow_spec.rounds` matmuls (square `matmul_dim`, seeds derived from sha256(replay_id|matmul_id|role)), runs `execute_challenge`, stashes one self-contained per-matmul attestation per round (single-matmul Challenge + single-result Response), and emits a PowStreamEntry referencing each. - 2026-05-04: Task 6.4 — verifier-side verdict on replay evidence. pkg/proverdet/replay_verify.py.verify_evidence(req, ev, *, fetch_attestation, backend) -> VerdictResult(passed, reasons[]) checks four things in parallel (no short-circuit, so the verdict carries every reason at once): erasure rounds count + passed count, pow_stream cadence vs requested rounds, sha256(decoded bytes_b64) == output.commitment, and per-round Freivalds rebuilt from output bytes (slice the concatenated C, build a new Response with stored A/B digests + the extracted C, run verify_response). Honest evidence passes; flipping a byte in output.bytes_b64 is caught because the Freivalds matvec disagrees on the spliced C; missing pow_stream is caught by the cadence check; unknown attestation IDs surface as "Freivalds: attestation ... not found"; downgraded erasure passed count surfaces as "erasure: only X/Y". Wired into the scheduler: ProverClient gained get_attestation; HttpProverClient adds GET /attestation/<id> (returns None on 404, raises on other 4xx/5xx); _do_replay collects the evidence chunk, parses ReplayRequest+ReplayEvidence, runs verify_evidence, and appends a /replay/verdict transcript entry (status 200 pass / 422 fail). DoD met: integration test asserts at least one /replay/verdict entry with status 200 lands when running the verifier against an unmodified prover.
- 2026-05-04: Task 7.1 — adversarial mixed_lora workload. experiments/prover-verifier-demo/scripts/workloads/mixed_lora.py.MixedLoraWorkload runs benign-shape inference (10 frames * 256 bytes per prompt + one record_task per prompt) and then performs `gradient_steps` extra simulated matmuls of dim `matmul_dim` per prompt that DO NOT call ctx.record_task. Each gradient step bumps an `observed_flops_total` counter by 2 * dim^3 (one matmul of square dim D ≈ 2D^3 flops). gradient_steps=0 ⇒ honest baseline (observed == claimed). gradient_steps>0 ⇒ observed - claimed = inferences * gradient_steps * 2 * dim^3 — exactly the discrepancy Phase 8.2's compute-budget signal will flag. The use_vllm branch is gated for the eventual GPU path (raises NotImplementedError on CPU). Tests: 6 cases covering claimed-only-inference, frames-only-inference, headline gap formula at dim=64 / gradient_steps=2, zero-step honest baseline, frame determinism, stop_event respect.
- 2026-05-04: Task 7.2 — adversarial lora_loading workload. experiments/prover-verifier-demo/scripts/workloads/lora_loading.py.LoraLoadingWorkload first GETs `lora_bytes` from `lora_url` (synthetic test path uses a tiny in-test ThreadingMixIn HTTPServer serving `b'L'*N`), republishes the bytes as outgoing frames in 256-byte chunks WITHOUT calling record_task, and only then runs benign-shape inference. The cheating signature: published bytes-on-wire = lora_bytes + (inferences * 10 * 256), but sum(claimed_flops) only accounts for the inferences — the gap (= lora_bytes) is what Phase 8.3's bandwidth signal flags. `download_bytes` is mutated during run() so tests can assert the knob round-trips. 6 unit tests including the required `test_download_bytes_match_knob` plus a `test_unattributed_bytes_equals_download_bytes` smoke check.

- 2026-05-04: Task 8.1 — replay-correctness signal. pkg/proverdet/verdict.py.SignalResult(passed, reasons) is the shared shape across Phase 8 signals; replay_correctness(transcript_entries) reads every received /replay/verdict/{replay_id} entry and returns passed=False with a per-replay reason if any has status_code != 200. To make replay_id available without touching transcript-payload bytes (the schema only stores payload_digest), the scheduler now records verdicts under endpoint /replay/verdict/{replay_id} instead of /replay/verdict. emit_verdict now returns "training_or_exfil" with the reasons concatenated when correctness fails, or "unknown" otherwise — Phase 8.3 will replace the unknown branch with a real two-signal combiner. Three unit tests cover all-pass / one-fail-with-id / no-verdicts-yet. Updated test_verifier_scheduler_against_prover.py to match the new `/replay/verdict/{replay_id}` endpoint convention.
- 2026-05-04: Task 8.2 — compute-budget signal. pkg/proverdet/verdict.py.compute_budget(summaries, *, tolerance=0.10) sums claimed_flops_total across all `kind=graph` summaries and observed_flops across all `kind=replay_evidence` summaries; fails iff observed > (1+tolerance) * claimed (equality at the boundary passes — predicate uses `>`). Special case: claimed=0 with observed>0 fails (cheating against the empty-graph baseline). The transcript only stores payload digests, so the scheduler now writes a sidecar summaries.jsonl alongside transcript.jsonl: each /graph received emits {kind:graph, claimed_flops_total, task_count}; each /replay evidence emits {kind:replay_evidence, replay_id, observed_flops=sum(2*dim^3*rounds), rounds, matmul_dim}. emit_verdict now reads transcript + summaries from the same dir and runs both replay_correctness and compute_budget — failing either fires "training_or_exfil"; only when no signal trips do we emit "unknown". Six unit tests cover under-claim / over-claim / boundary equality / off-by-one above tolerance / empty input / zero-claim-with-positive-observed.
- 2026-05-04: Task 8.3 — bandwidth signal + binary verdict combiner. pkg/proverdet/verdict.py.bandwidth_signal(traffic_size, claimed_artifact_bytes, *, tolerance=0.10) fails iff traffic_size > (1+tolerance)*claimed_artifact_bytes (claim<=0 ⇒ insufficient baseline ⇒ pass and rely on the other signals). The benign workload sets claimed_flops == bytes-on-wire by construction, so emit_verdict reuses the summed claimed_flops as the bandwidth baseline; lora_loading's downloaded bytes are unattributed (no record_task) so they appear as a positive gap. compute_budget gained a workload_summary kwarg threading the prover's /workload/stop response (claimed_flops_total + observed_flops_total) into the same predicate — necessary because mixed_lora's gradient-step cheating happens entirely inside the workload thread and never surfaces in the scheduler's /replay challenges. The combiner is binary: any failed signal ⇒ "training_or_exfil" with reasons concatenated; all passed ⇒ "inference" (dropped "unknown" — the headline output is binary). pkg/proverdet/workload_runner.WorkloadRunner.stop now returns int (the workload's observed_flops_total via duck-typed getattr); cmd/prover/main.py._handle_post_workload_stop returns {stopped, claimed_flops_total, observed_flops_total, task_count} and ProverState gained a public task_totals() snapshot to keep the handler off the private _tasks_lock. cmd/verifier_cli/main.py accepts --traffic-digest and --workload-summary; the e2e adversarial test now POSTs /workload/stop, writes its body to workload_summary.json, passes both flags through, and asserts verdict == "training_or_exfil" with non-empty reasons. Five new combiner unit tests cover honest baseline (passes) + each failing signal in isolation (replay status, workload-summary compute budget, bandwidth) + multi-signal failure (reasons concatenate). The pre-Phase-8.3 stub tests were updated from "unknown" to "inference" to match the new binary contract.
