- 2026-05-04: Task 2.2 — added GET /graph returning empty placeholder. pkg/proverdet/graph_builder.py.build_empty_graph(run_id) is the pure constructor; the handler validates the body against schema before serialization (500 if mismatch — programmer error).

- 2026-05-04: Task 2.4 — added prover-side capture log. pkg/proverdet/capture.py.ProverCaptureLog writes monotonic-seq canonical-JSON entries with sha256-prefixed payload digests; threadsafe under 4-thread × 50-entry stress. - 2026-05-04: Task 3.4 — verdict CLI stub. cmd/verifier_cli/main.py reads --transcript, calls pkg/proverdet/verdict.emit_verdict (stub returns {"verdict":"unknown","reasons":[]}), writes canonical-JSON --out. Phase 8 will replace the stub with real signals; the CLI signature stays compatible.
- 2026-05-04: Task 4.1 — prover traffic publisher. pkg/proverdet/traffic_publisher.py.TrafficPublisher batches frames in a background thread and POSTs to the verifier's /traffic. publish() never blocks the caller; stop() drains and joins. Tests use a tiny in-test recording server to assert byte-for-byte concatenation across 100 frames.
- 2026-05-04: Task 4.2 — verifier traffic ingest finalizer. /traffic appends to traffic.bin and updates a running sha256; /traffic/finalize is idempotent (returns the same digest+size on repeat calls) and writes traffic.digest. POSTing to /traffic after finalize returns 409. Both endpoints are recorded in the transcript.

- 2026-05-04: Task 5.1 — concrete WorkloadContext (no ABC). Three callable fields: publish_frame, record_task, stop_event. The first concrete workload (5.2) defines the duck-typed shape; later workloads call the same methods. Honors the "don't build a BaseWorkload ABC" rule.
- 2026-05-04: Task 5.2 — benign inference workload. CPU-friendly synthetic frames (10 frames × 256 bytes per prompt, deterministic given (prompt, seed)); use_vllm=True branch lazy-imports vllm and is gated for GPU validation. Tests assert frame count, frames-per-prompt determinism, task records, and stop_event respect.
- 2026-05-04: Task 5.3 — wired /workload/start and /workload/stop. pkg/proverdet/workload_runner.py.WorkloadRunner is the at-most-one thread manager (literal name→class dict — no entry points). Tests cover happy path, 409 on second start, 404 on unknown name, 400 on missing name. Added benign workload's delay_per_prompt_s knob so the 409 test can keep one workload deliberately alive while the other tries to start.
- 2026-05-04: Task 6.1 — replaced stub /replay with Freivalds-backed evidence. pkg/proverdet/attestation_store.py is a threadsafe in-memory dict (lifetimes match the prover process). pkg/proverdet/replay.py.produce_evidence builds a Challenge of `pow_spec.rounds` matmuls (square `matmul_dim`, seeds derived from sha256(replay_id|matmul_id|role)), runs `execute_challenge`, stashes one self-contained per-matmul attestation per round (single-matmul Challenge + single-result Response), and emits a PowStreamEntry referencing each. Output commitment is sha256 of the concatenated C bytes; this lets the verifier (Task 6.4) re-run Freivalds against each /attestation/{id} and check that the per-matmul outputs hash to the request's commitment. cmd/prover/main.py adds GET /attestation/{id} and switches /replay to produce_evidence. Wire dtype maps: int8 → bitwise int8/int32; fp16/bf16 → tolerance with fp32 acc (only int8 works on stdlib backend, hence test+default scheduler now use dtype=int8, matmul_dim=8). Tampering test flips a byte of the stored c_b64 and asserts verify_response returns overall_passed=False.
- 2026-05-04: Task 6.2 — wired PoSE-style erasure evidence into /replay. pkg/proverdet/erasure.py implements an HMAC-SHA256(seed, round_index)-based protocol with a Backend Protocol; HmacErasureBackend is the honest path. run_erasure(spec, log_path, backend) writes one JSONL ErasureRoundLog per round and returns ErasureEvidence with rounds=spec.rounds, passed=count(ok). verify_round_log replays the seed and rejects tampered entries (used by Phase 6.4). Tests cover honest-all-pass, JSONL shape, log-replay symmetry, dishonest backend (returns 32 zero bytes — fewer rounds pass than total), and tampered-log rejection. Replaced the static `erasure_evidence=ErasureEvidence(...passed=rounds)` filler in produce_evidence with a real run; added an `erasure_log_dir` kwarg so the prover server writes to <out_dir>/erasure/erasure-<replay_id>.jsonl. Didn't import the full PoSE pipeline — its modules expect a real GPU node; we lifted the protocol logic only.
- 2026-05-04: Task 6.3 — streaming PoW evidence (NDJSON, contract-breaking). pkg/proverdet/replay.py.produce_evidence_stream is a generator yielding {"kind":"pow",...PowStreamEntry} per matmul as it completes, then exactly one final {"kind":"evidence",...ReplayEvidence}. produce_evidence is now a thin sync wrapper that consumes the stream and returns the final evidence. cmd/prover/main.py._handle_post_replay sends 200 with Content-Type: application/x-ndjson and Connection: close (stdlib BaseHTTPRequestHandler doesn't support chunked encoding cleanly; client reads to EOF), pre-flight-checks dtype via check_supported() so 4xx still surfaces synchronously, and writes one capture-log entry covering the full streamed payload. pkg/proverdet/scheduler.HttpProverClient.post_replay is now an Iterator[tuple[int, dict]]; the scheduler records each chunk as its own transcript entry. tests/integration/test_replay_streaming.py asserts (3 pow lines, 1 evidence line, evidence comes last) and that pre-stream 400 still works. Updated _helpers.http_post_ndjson and migrated test_prover_replay_endpoint.py + test_prover_attestation_endpoint.py to it. Existing scheduler unit FakeClient was rewritten to yield NDJSON chunks instead of returning a single body.
